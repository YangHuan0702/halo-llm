# 章节 02：Prompt、API 与结构化输出

## 学习目标

- 学会写可复用 Prompt，而不是临时对话。
- 理解 system prompt、few-shot、约束输出、JSON schema 的作用。
- 学会让模型稳定返回机器可消费的结构。

## 核心知识

### 1. Prompt 的层次

- System Prompt：定义身份、边界和全局规则。
- User Prompt：具体任务。
- Context Prompt：业务上下文、知识片段、检索结果。
- Output Constraint：格式要求、字段约束、禁止项。

### 2. 高质量 Prompt 的共性

- 任务边界清晰
- 输入字段明确
- 输出格式固定
- 失败时行为可预期
- 示例足够但不过载

### 3. 为什么结构化输出重要

- 便于后端解析
- 便于自动化评测
- 便于接入工作流和 Agent
- 能降低“答了一堆却无法落库”的问题

### 4. 常见策略

- JSON 模式输出
- 函数调用 / 工具调用
- Pydantic / Schema 校验
- 自修复重试

## 实操建议

1. 设计一个“文本分类” Prompt。
2. 设计一个“信息抽取” Prompt。
3. 让模型严格输出 JSON。
4. 对格式错误做一次自动重试。

## 完成标准

- 你能解释 Prompt 模板和临时对话的区别。
- 你能让模型以稳定 JSON 输出结果。
- 你知道为什么“看起来格式像 JSON”不等于可直接解析。
