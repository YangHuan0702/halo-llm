# 章节 11：LLM 全链路监控与可观测性

## 学习目标

- 理解 LLM 系统要监控的不只是 CPU 和内存。
- 学会观测 Prompt、检索、工具调用、模型输出、延迟、成本、错误率。
- 认识 tracing、logging、metrics 三者在 LLM 系统中的作用。

## 理论重点

### 1. 可观测性三件套

- Logs：离散事件和错误细节
- Metrics：聚合指标
- Traces：跨组件调用链

### 2. LLM 系统特有指标

- Token 输入输出量
- 模型耗时
- 工具调用成功率
- 检索命中率
- Prompt 版本
- 单请求成本
- 用户反馈分

### 3. 为什么全链路重要

- 答错可能不是模型问题
- 可能是：
  - 检索没召回
  - 工具超时
  - Prompt 版本回退
  - 上下文拼接错误

### 4. 常见工具方向

- Langfuse
- Phoenix
- OpenTelemetry
- Prometheus
- Grafana

## 本章实操

1. 给一个现有系统加 tracing。
2. 记录 prompt、模型、延迟、token、错误。
3. 设计一个简单监控看板。

## 完成标准

- 你能定位一次“回答差”的根因，不只是说模型不行。
- 你能说出哪几个指标最值得长期看。
