# 章节 10：自动化评测流水线

## 学习目标

- 理解没有评测就没有优化闭环。
- 学会构建数据集、指标、基线、自动化回归测试。
- 掌握 LLM-as-a-Judge、规则评测、人工抽检之间的组合思路。

## 核心知识

### 1. 为什么 LLM 系统更需要评测

- 输出非确定
- 业务目标复杂
- 小改动可能带来大回归

### 2. 评测对象

- 提示词
- 检索效果
- 工具调用正确率
- Agent 任务完成率
- 延迟
- 成本
- 稳定性

### 3. 评测方法

- 规则评测
- 精准答案对比
- 人工标注
- LLM-as-a-Judge
- A/B 测试

### 4. 评测流水线组成

- 测试集
- 基线版本
- 评测脚本
- 指标存储
- 报告输出
- CI 或定时执行

## 本章实操

1. 为你的 RAG 或 Agent 系统建立 30 条测试样本。
2. 定义至少 3 个指标。
3. 做一版自动评测脚本。
4. 输出对比报告。

## 完成标准

- 你能明确说明什么叫“变好了”，什么叫“退化了”。
- 你能解释自动评测和人工评测各自的盲点。
