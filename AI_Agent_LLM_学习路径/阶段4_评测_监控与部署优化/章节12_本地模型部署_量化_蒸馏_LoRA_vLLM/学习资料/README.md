# 章节 12：本地模型部署、量化、蒸馏、LoRA 与 vLLM

## 学习目标

- 了解为什么很多团队会考虑本地部署小模型。
- 掌握量化、蒸馏、LoRA 微调、推理加速的基本概念和适用边界。
- 理解 vLLM 的核心价值，以及它在推理服务中的常见定位。

## 理论重点

### 1. 为什么要本地部署

- 降低长期调用成本
- 数据不出域
- 更可控的延迟
- 可做定制化微调

### 2. 小模型部署的现实约束

- 显存
- 吞吐
- 并发
- 上下文长度
- 模型能力上限

### 3. 量化

- 目标：降低显存占用、提升部署可行性
- 代价：可能损失精度
- 常见思路：4bit、8bit

### 4. 蒸馏

- 用强模型能力迁移给小模型
- 目标是更低成本保留关键能力
- 前提是要有较高质量的数据与目标任务定义

### 5. LoRA / PEFT

- 以较小参数增量完成任务适配
- 适合领域风格、分类、抽取、格式化等特定任务增强
- 不适合把一个很弱的底座直接“调成万能模型”

### 6. vLLM

- 主要价值：提升推理吞吐、优化显存利用、支持高并发服务
- 典型关键词：
  - continuous batching
  - paged attention
  - 高吞吐 serving

### 7. 选型思路

- 低并发验证期：先 API
- 数据敏感且高频：评估本地部署
- 有领域任务：考虑 LoRA
- 并发高：评估 vLLM
- 显存受限：优先量化

## 本章实操

1. 本地部署一个小模型。
2. 比较量化前后资源占用与效果。
3. 跑一个简单 LoRA 任务。
4. 用 vLLM 暴露推理服务并测试吞吐。

## 完成标准

- 你能解释量化、蒸馏、LoRA、vLLM 的区别。
- 你能说明它们分别解决的是哪个工程问题。
