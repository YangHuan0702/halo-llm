# 章节 06：多 Agent、规划与人机协同

## 学习目标

- 理解多 Agent 不只是“多开几个模型”。
- 学会设计角色分工、协作协议、共享状态和人工介入点。
- 能判断一个问题是否真的需要多 Agent。

## 理论重点

### 1. 多 Agent 的价值

- 分工明确
- 便于复杂任务拆解
- 适合评审、计划、执行分离

### 2. 常见角色

- Planner：拆解任务
- Researcher：收集信息
- Executor：执行动作
- Reviewer：质量评审
- Human：人工审批或纠偏

### 3. 何时不该用多 Agent

- 任务很短
- 工具很少
- 时延要求高
- 成本预算紧

### 4. 人机协同

- 高风险动作前必须人工确认
- 模型给建议，人类做最终授权
- 审批节点要有上下文和依据

## 实操建议

1. 设计一个 Planner + Executor 双 Agent 流程。
2. 增加 Reviewer 节点，对输出做二次检查。
3. 在关键步骤加入人工确认。

## 完成标准

- 你能说清“多 Agent 的收益”和“多 Agent 的额外复杂度”。
- 你能设计一个明确的消息协议或共享状态结构。
