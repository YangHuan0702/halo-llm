# 章节 05：单 Agent、多工具与记忆

## 学习目标

- 理解 Agent 和普通链式调用的根本差异。
- 学会构建“模型做决策，工具做执行”的基本架构。
- 理解短期记忆、长期记忆和外部状态存储的区别。

## 理论重点

### 1. 什么是 Agent

- 不是单次问答
- 而是：目标驱动 + 动态决策 + 工具调用 + 状态更新

### 2. Agent 的最小闭环

- 接收目标
- 判断下一步
- 调用工具
- 读取结果
- 再判断是否继续

### 3. 工具设计原则

- 工具职责单一
- 输入输出明确
- 错误信息可读
- 幂等或可恢复

### 4. 记忆类型

- 短期记忆：当前会话上下文
- 长期记忆：用户画像、偏好、历史任务
- 外部状态：数据库、缓存、文件、向量库

### 5. 为什么很多 Agent Demo 不可用

- 工具描述含糊
- 没有状态约束
- 缺少终止条件
- 没有成本和延迟意识

## 实操建议

1. 实现两个工具：
   - 搜索工具
   - 计算工具
2. 做一个 Agent，根据问题决定调用哪个工具。
3. 为 Agent 增加一次会话记忆。

## 完成标准

- 你能区分“调用工具”和“让模型假装自己会”。
- 你能描述 Agent 循环中的终止条件。
