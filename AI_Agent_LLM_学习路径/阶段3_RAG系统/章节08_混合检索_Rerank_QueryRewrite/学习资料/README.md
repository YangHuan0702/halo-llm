# 章节 08：混合检索、Rerank 与 Query Rewrite

## 学习目标

- 理解为什么纯向量检索不够。
- 掌握 BM25、向量检索、混合检索、Reranker、查询改写之间的关系。
- 学会为真实业务问题提高命中率和排序质量。

## 理论重点

### 1. 混合检索

- 关键词检索擅长精确术语匹配
- 向量检索擅长语义相似
- 混合检索是为了兼顾二者

### 2. Rerank 的作用

- 第一阶段先多召回
- 第二阶段再精排
- 本质是在“召回率”和“精确率”之间做平衡

### 3. Query Rewrite

- 把用户口语问题改写成更适合检索的查询
- 适合处理：
  - 指代不明
  - 关键词缺失
  - 多跳问题

### 4. 典型失败模式

- 关键词太强，语义弱
- 语义太宽，召回太散
- 改写过度，偏离用户原意
- Rerank 模型和业务领域不匹配

## 本章实操

1. 实现 BM25 + 向量检索混合召回。
2. 加一个 reranker。
3. 针对 20 个问题对比优化前后效果。

## 完成标准

- 你能用数据说明混合检索是否真的提升了结果。
- 你能指出 Query Rewrite 的收益和风险。
