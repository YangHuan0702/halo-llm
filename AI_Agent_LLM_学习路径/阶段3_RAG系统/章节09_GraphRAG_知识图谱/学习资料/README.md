# 章节 09：GraphRAG 与知识图谱

## 学习目标

- 理解 GraphRAG 想解决什么问题。
- 认识实体、关系、社区、子图、路径检索等核心概念。
- 能用图结构增强复杂关系问答或多跳问答。

## 理论重点

### 1. 为什么会有 GraphRAG

- 文本块相似，不代表结构关系清晰
- 多跳问题、实体关系推理、跨文档关联问题，图结构更自然

### 2. GraphRAG 基本链路

- 文档抽取实体和关系
- 构建图谱
- 做社区划分或聚类
- 查询时定位相关实体和子图
- 组装上下文给模型回答

### 3. GraphRAG 的代价

- 构图成本高
- 实体消歧难
- 关系抽取质量影响很大
- 工程复杂度明显高于基础 RAG

### 4. 适用场景

- 产业研究
- 医疗知识关联
- 法律法条关联
- 企业组织知识图谱

## 本章实操

1. 从一批文档中抽取实体和关系。
2. 构建一个小型知识图谱。
3. 实现一个简单的图增强问答流程。

## 完成标准

- 你能解释 GraphRAG 相对传统 RAG 的收益和成本。
- 你能举出至少一个必须依赖关系链路才能答好的问题。
