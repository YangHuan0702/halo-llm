# 章节 07：基础 RAG、索引与召回

## 学习目标

- 理解为什么需要 RAG，而不是只依赖模型参数记忆。
- 掌握文档切分、向量化、索引、检索、上下文注入的基础流程。
- 能构建一个最小可用 RAG 问答系统。

## 核心知识

### 1. RAG 的基本链路

- 文档采集
- 清洗与切分
- 向量化
- 建索引
- 查询改写
- 检索召回
- 上下文组装
- 模型生成

### 2. Chunking 为什么关键

- 切太大：召回不精准
- 切太小：语义不完整
- 重叠太多：冗余高、成本高

### 3. Embedding 与相似度

- Embedding 是语义向量表示
- 常见相似度：余弦相似度、点积
- 向量检索不等于事实正确

### 4. RAG 常见失败点

- 文档质量差
- 分块策略错误
- 召回不足
- 上下文污染
- 模型没遵守“仅基于材料回答”

## 本章实操

1. 准备 20 到 50 篇文档。
2. 切分后写入向量库。
3. 实现问答接口。
4. 返回答案时附带命中文档片段。

## 完成标准

- 你能解释为什么 RAG 的瓶颈常常不在模型，而在检索链路。
- 你能展示至少一次“错误分块导致错误答案”的案例。
