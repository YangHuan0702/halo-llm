# 提交流程

你完成某一章节后，按下面格式把内容交给我评分。

## 你需要提交的内容

1. 理论考试答案
2. 代码仓库或关键文件路径
3. 运行说明
4. 项目结果截图或日志
5. 复盘总结

## 推荐提交模板

```text
章节：

理论考试答案：
1.
2.
3.

代码考试说明：
- 入口文件：
- 运行命令：
- 实现要点：

实际项目说明：
- 项目目标：
- 技术栈：
- 已完成功能：
- 演示方式：
- 遇到的问题与解决：

自评：
- 你认为最强的一点：
- 你认为最弱的一点：
```

## 我评分后的输出格式

我会按下面结构返回：

1. 总分
2. 理论考试得分与原因
3. 代码考试得分与原因
4. 实际项目得分与原因
5. 必须修正项
6. 是否允许进入下一章节
