# 章节 04：Dify 与 n8n 自动化编排

## 学习目标

- 理解低代码 / 可视化编排平台在企业中的价值与边界。
- 学会用 Dify 做一个基础 AI 应用，用 n8n 串联外部系统。
- 认识“产品验证速度”和“工程可控性”之间的权衡。

## 核心知识

### 1. Dify 适合什么

- 快速构建对话应用
- Prompt 管理
- 知识库接入
- 简单工作流

### 2. n8n 适合什么

- 系统间自动化
- 事件驱动流程
- 表单、Webhook、数据库、第三方 API 编排
- AI 节点和业务节点混搭

### 3. 为什么两者经常配合

- Dify 负责 AI 应用侧能力
- n8n 负责业务系统联动
- 二者结合可以快速做 MVP

### 4. 风险与边界

- 低代码不等于低复杂度
- 可视化流程一旦变大，维护成本会上升
- 核心逻辑和关键数据校验仍需要代码层兜底

## 本章实操

1. 用 Dify 做一个知识问答或表单分析应用。
2. 用 n8n 接一个 Webhook。
3. 让 n8n 调用 LLM，然后把结果写回数据库或表格。

## 完成标准

- 你能解释 Dify、n8n 和纯代码方案各自的适用边界。
- 你能做出一个可运行的自动化流程。
